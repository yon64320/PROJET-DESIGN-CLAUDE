## A) TONE & TERMINOLOGY GUIDE

### Voice

**3 adjectifs** : Direct, Chaleureux, Factuel

- **Direct** : chaque phrase va au point. Pas de périphrases, pas de tournures corporate. Le commerçant lit entre deux clients — il a 8 secondes.
- **Chaleureux** : on parle à des humains qui nourrissent leur quartier, pas à des DSI. Le vouvoiement est systématique, mais le ton reste proche, jamais guindé.
- **Factuel** : chaque affirmation s'appuie sur un chiffre, un mécanisme ou une preuve. Zéro filler.

**Ce que le ton N'EST PAS** :

- Pas corporate ("Nous vous accompagnons dans votre transformation digitale")
- Pas startup cool ("Révolutionnez votre business en un clic !")
- Pas condescendant ("Même vous pouvez y arriver")
- Pas pompeux ("Fort de notre expertise reconnue")
- Pas familier (pas de tutoiement, pas d'argot)

### Niveau de preuve attendu

Chaque claim du site doit passer le test :

1. **Est-ce vérifiable ?** → Si oui, citer le chiffre ou la source.
2. **Est-ce mécanistique ?** → Si oui, décrire COMMENT ça marche en 1 phrase.
3. **Est-ce restrictif ?** → Si oui, dire POUR QUI / PAS POUR QUI.

Si aucune de ces trois conditions n'est remplie → le claim est du filler. Le supprimer ou le marquer `[PROOF NEEDED]`.

### Règles microcopy CTA

- **Verbe d'action** en premier : "Demander", "Découvrir", "Voir", "Envoyer"
- **Promesse** intégrée quand possible : "Demander une démo" > "Nous contacter" (la démo est une promesse concrète)
- **Réduction de friction** en micro-texte sous le CTA : "Rappel sous 24h", "Sans engagement", "15 minutes"
- **Cohérence** : un même CTA = un même libellé partout sur le site (pas "Demander une démo" sur une page et "Réserver ma démo" sur une autre)
- **Vouvoiement** dans les CTA personnalisés : "Envoyer ma demande" (pas "Envoyez votre demande")

## PACKAGE FOR UI/TECH

### Contraintes de mise en page

1. **Bilingue FR/EN** : tous les textes sont livrés dans les deux langues. Le texte anglais est plus court que le français en moyenne — les composants doivent supporter les deux longueurs sans casser le layout.

2. **Hiérarchie Hn stricte** : chaque page a 1 seul H1. Les sections sont en H2. Les sous-sections (blocs de bénéfices, paliers) en H3. Pas de H4+ sauf exception justifiée.

3. **Pas de prix** : les 4 paliers (Essentiel, Smart, Marketing, Analytics) sont présentés par fonctionnalités UNIQUEMENT. Aucun emplacement pour un prix ne doit être prévu dans le design des cards tarifs.

4. **Double formulaire Contact** : la page Contact contient DEUX formulaires visuellement distincts sur la même page. L'un pour les commerçants (5 champs + 1 optionnel), l'autre pour les distributeurs (4 champs). Ils ne doivent pas être confondus.

5. **Témoignages = placeholders** : les sections témoignages sont prévues mais le contenu est `[À FOURNIR]`. Le design doit prévoir l'emplacement (photo + citation + nom/métier/ville) mais un fallback visuel est nécessaire tant que les témoignages ne sont pas disponibles.

6. **CTA minimum 2 par page** : hero + pré-footer. Le CTA "Demander une démo" doit être identique visuellement partout.

7. **Bouton WhatsApp flottant** : sticky bas-droite, toutes pages, toutes tailles d'écran. Numéro `[À FOURNIR]`.

8. **Logo bar** : ~50 logos HubRise en B&W, scroll horizontal infini, CSS-only, pas de hover.

9. **Smartphone 3D** (hero Accueil) : affiche une conversation WhatsApp. Fallback = image statique + alt text fourni. Loading state + hints d'interaction fournis.
