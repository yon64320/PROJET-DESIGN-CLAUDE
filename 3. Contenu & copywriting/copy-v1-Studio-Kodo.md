## A. TONE & TERMINOLOGY GUIDE

### A.1 Voice

**3 adjectifs :** Confiant · Direct · Raffiné

- **Confiant** : Studio Kodo affirme, ne suggère pas. Pas de "nous pensons que", pas de conditionnel. Le travail parle.
- **Direct** : Phrases courtes. Sujet-verbe-objet. Pas de périphrases. Chaque mot a un rôle.
- **Raffiné** : Le vocabulaire est précis, jamais technique. L'élégance vient de la sobriété, pas de l'emphase.

**Ce que le ton N'EST PAS :**
- Arrogant (confiant ≠ prétentieux)
- Corporate froid (direct ≠ robotique)
- Familier/casual (raffiné ≠ décontracté)
- Technique/jargon (direct ≠ acronymes)

**Mode d'adresse :** Vouvoiement systématique.

### A.2 Niveau de preuve attendu

Chaque affirmation doit être adossée à :
- une **preuve** (case study, chiffre, timeline), OU
- un **mécanisme** (comment c'est possible), OU
- une **contrainte** (pour qui / pas pour qui).

Pas de claim flottant. Pas de "qualité premium" sans démonstration.

### A.3 Lexique

**Mots à utiliser :**

| Mot / Expression | Raison |
|------------------|--------|
| Sur-mesure | Opposition directe au template |
| Livré | Concret, factuel — résultat tangible |
| Réactif | Contraste avec la lenteur agence |
| Votre / Vous | Centré client |
| Résultat | Orienté outcome |
| Concret | Ancre dans le réel |
| 3 jours | Le chiffre le plus différenciant |
| Interlocuteur unique | Mécanisme clair |

**Mots interdits :**

| Mot / Expression | Raison |
|------------------|--------|
| Synergie | Jargon corporate creux |
| Disruption | Cliché startup |
| Game-changer | Hyperbole invérifiable |
| Passion / passionné | Galvaudé |
| "Notre équipe" | Freelance solo — ne pas mentir |
| Accompagnement 360° | Vide de sens |
| Solution | Trop vague |
| Innovant | Auto-décerné, invérifiable |
| Expérience utilisateur unique | Cliché web design |

### A.4 Règles CTA / microcopy

- **Verbes d'action** en premier : "Demander", "Voir", "Découvrir", "Démarrer"
- **Promesse dans le CTA** quand possible : "Demander un devis" > "Envoyer"
- **Réducteur de friction** à proximité du CTA : "Pas d'engagement", "1min30", "Rappel sous 24h"
- **Jamais de majuscules intégrales** (pas de "DEMANDER UN DEVIS")
- **Ton des messages d'erreur** : utile, pas culpabilisant. Expliquer pourquoi le champ est nécessaire.

#### Section 2 — Brief

**H2 :**
> Le brief

> Maison Levain est une boulangerie artisanale installée dans le centre d'une ville moyenne. Pains au levain, viennoiseries maison, farines locales — un savoir-faire remarquable, une clientèle fidèle.
>
> Le problème : aucune présence en ligne. Pas de logo cohérent, pas de site, pas de visibilité au-delà du quartier. Les nouveaux arrivants en ville ne savaient pas que Maison Levain existait. Les commandes de pains spéciaux se faisaient par téléphone, sans système.
>
> L'objectif : une identité visuelle complète et un site vitrine qui reflètent la qualité du produit — et qui permettent aux clients de commander en ligne pour le retrait en boutique.

#### Section 3 — Solution & services

**H2 :**
> La solution

**Services appliqués :** Branding complet + Site vitrine premium

> **Branding** — Direction artistique inspirée de l'univers mid-century : tons chauds (beurre, terre cuite, crème), typographies géométriques évoquant les enseignes d'époque, logo monogramme "ML" déclinable de la devanture au papier d'emballage.
>
> **Site vitrine** — 5 pages (Accueil, Notre pain, La boutique, Commander, Contact). Photographies traitées en mode éditorial — le pain comme objet d'art. Formulaire de commande intégré (choix du pain, quantité, créneau de retrait). Responsive, rapide, accessible.
>
> **Ce qui rend le projet singulier** : le branding ne s'est pas limité au digital. La charte a été pensée pour vivre en boutique (étiquettes, affiches, packaging) autant que sur écran. Cohérence totale entre l'expérience physique et l'expérience en ligne.

| **Jour 3** | Design UI | Maquettes du site (desktop + mobile). Traitement photographique défini. Formulaire de commande conçu. |

**Règle technique :** Si `prefers-reduced-motion: reduce` est détecté, les animations GSAP sont réduites via `gsap.matchMedia()`. Le Spline 3D island affiche l'image de fallback statique. **Règle copy :** Aucun message visible n'est nécessaire quand le système respecte automatiquement `prefers-reduced-motion`. Le toggle n'est proposé que si l'Agent 4/5 décide d'un contrôle utilisateur explicite.

## PACKAGE FOR UI/TECH

### Contraintes layout issues du copy

1. **Hero Accueil** — H1 court (6 mots), 1 sous-titre, 2 CTA côte à côte. **Spline 3D island** en arrière-plan ou en zone dédiée (lazy-loaded, WebGL detection + fallback image statique). Scroll indicator animé en bas du hero. Magnetic buttons sur les CTA.
2. **Bande de différenciation** — 3 blocs compacts en ligne (desktop) / empilés (mobile). Icônes + titre bold + 1 phrase.
3. **Grille réalisations** — 4 vignettes format éditorial large. Badge "Projet concept" superposé. Hover state.
4. **Cartes services** — 3 cartes avec scope en bullets. Pas de tableau pricing. CTA par carte.
5. **Process** — 3 étapes visuelles type frise. Chaque étape : titre + texte + durée.
6. **Case studies** — Template identique ×4. Sections verticales : hero full-bleed → brief → solution → timeline (tableau) → galerie → résultats (gros chiffres) → réflexion → CTA → nav.
7. **Formulaire multi-étapes** — 5 étapes, 1 écran par étape, barre de progression, boutons retour/suivant. Layout centré, pas de sidebar (desktop → réassurance en colonne latérale).
8. **À propos** — Layout asymétrique texte + photo. 4 blocs convictions. Frise process détaillée.
9. **Footer** — 2 lignes : nav + légales + social. Copyright en dessous.
10. **Tier 3 global** — Word-split text sur H1/H2 principaux (GSAP SplitText). Parallax sur les sections visuelles (case studies, hero). Smooth scroll (Lenis) global. Magnetic effect sur tous les CTA primaires. Skeleton state pour le Spline 3D island. Tous les effets doivent se dégrader proprement avec `prefers-reduced-motion`. Voir section C.11 pour le microcopy complet.
